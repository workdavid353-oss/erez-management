1 דרישות מקדימות
התקן Docker Desktop ידני
הורד מ-https://www.docker.com/products/docker-desktop והתקן.
לאחר ההתקנה — הפעל מחדש את המחשב ווודא ש-Docker פועל (אייקון במגש המערכת).
⚠️ ברשת TAU — Docker Desktop עלול להתקשות בהורדת images בפעם הראשונה. יש להיות ברשת Wi-Fi ביתית לשלב ההתקנה הראשון.
התקן Node.js 20+ ידני
הורד מ-https://nodejs.org (LTS). לאחר ההתקנה בדוק: 
node --version
npm --version
התקן Git ידני
הורד מ-https://git-scm.com והתקן עם הגדרות ברירת מחדל.
2 הרץ את סקריפט ההתקנה האוטומטי
הרץ setup.ps1 אוטומטי
פתח PowerShell כמנהל מערכת והרץ: 
powershell -ExecutionPolicy Bypass -File "C:\path\to\erez-management\setup.ps1"
הסקריפט יבקש ממך:
  • כתובת ה-IP של השרת ברשת המקומית
  • טוקן GitHub Actions (מהשלב הבא)
3 הגדרת Supabase
העתק תיקיית Supabase Docker ידני
העתק את התיקייה supabase\docker\ מהמחשב הישן לשרת החדש — הנתיב לא חייב להיות זהה, רק תזכור איפה שמת אותה.
💡 כל מה שצריך זה שהתיקייה תכיל docker-compose.yml ו-.env. תריץ את docker compose up -d מתוכה.
עדכן את ה-IP ב-supabase/docker/.env ידני
קודם כל מצא את ה-IP של השרת — הרץ ב-PowerShell: 
Get-NetIPAddress -AddressFamily IPv4 | Where-Object { $_.IPAddress -notlike '127.*' -and $_.IPAddress -notlike '169.*' -and $_.IPAddress -notlike '172.*' } | Select-Object IPAddress, InterfaceAlias
קח את ה-IP שמשויך לממשק הרשת שהעובדים מחוברים אליו (לרוב Ethernet או Wi-Fi).

ערוך את הקובץ supabase\docker\.env ועדכן:
מפתחערך
API_EXTERNAL_URLhttp://<LAN_IP>:8000
SITE_URLhttp://<LAN_IP>
💡 אותו IP תשתמש בו גם ב-setup.ps1 כשהוא ישאל אותך.
הפעל את Supabase אוטומטי
cd C:\Users\<user>\Desktop\VS-Code\supabase\docker
docker compose up -d
המתן ~2 דקות עד שכל הקונטיינרים עולים. בדוק ב: 
docker ps --filter "name=supabase"
כולם צריכים להיות healthy.
הרץ את ה-SQL — סכמה מלאה חד-פעמי
פתח Supabase Studio ב-http://localhost:3000 → SQL Editor → לחץ New Query.

העתק את כל התוכן של הקובץ supabase_schema.sql שנמצא בתיקיית הפרויקט, הדבק ב-editor ולחץ Run.
💡 הקובץ יוצר את כל הטבלאות, RLS policies, ו-triggers. חובה להריץ לפני שמשתמשים נכנסים לראשונה.
הרץ Migration — עמודת preferences חד-פעמי
באותו SQL Editor, פתח New Query נוספת והרץ: 
ALTER TABLE public.profiles
ADD COLUMN IF NOT EXISTS preferences JSONB NOT NULL DEFAULT '{}';
תקן משתמשים שנוצרו ידנית ב-SQL חד-פעמי
אם ייבאת משתמשים ישירות ל-DB (לא דרך ממשק הרשמה), הרץ: 
UPDATE auth.users
SET
  email_change  = COALESCE(email_change, ''),
  phone_change  = COALESCE(phone_change, '')
WHERE email_change IS NULL OR phone_change IS NULL;
⚠️ בלי זה תופיע שגיאת "Database error querying schema" בכניסה.
4 יצירת משתמשים
צור את המשתמש Admin הראשון חד-פעמי
בהתקנה ראשונה אין עדיין משתמשים — יש ליצור admin ראשון ידנית דרך Supabase Studio.

שלב א׳ — פתח http://localhost:3000 → Authentication → Users → Add User
מלא אימייל וסיסמה ולחץ Create User.

שלב ב׳ — עבור ל-SQL Editor והרץ (החלף את הערכים): 
INSERT INTO public.profiles (id, full_name, role)
SELECT id, 'שם מלא כאן', 'admin'
FROM auth.users
WHERE email = 'email@example.com';

-- תיקון שדות NULL (חובה למשתמשים שנוצרו ידנית)
UPDATE auth.users
SET
  email_change = COALESCE(email_change, ''),
  phone_change = COALESCE(phone_change, '')
WHERE email = 'email@example.com';
💡 החלף 'שם מלא כאן' ו-'email@example.com' בערכים האמיתיים.
יצירת משתמשים נוספים דרך הממשק אוטומטי
לאחר שה-admin הראשון מחובר — כל המשתמשים הבאים נוצרים דרך האפליקציה:

הגדרות → משתמשים → + הוסף משתמש

מלא שם מלא, אימייל, סיסמה ותפקיד (admin / עובד). המערכת יוצרת את המשתמש אוטומטית ב-Supabase.
⚠️ רק admin או owner יכולים ליצור משתמשים חדשים.
5 הגדרת GitHub Actions Runner
קבל טוקן Runner מ-GitHub ידני
פתח בדפדפן:
https://github.com/workdavid353-oss/erez-management/settings/actions/runners/new?runnerOs=windows

בחר Windows → x64, גלול ל-Configure וצלם את הטוקן שמופיע אחרי --token.
⚠️ הטוקן תקף רק שעה אחת!
הכנס את הטוקן לסקריפט ההתקנה אוטומטי
הסקריפט setup.ps1 מבקש את הטוקן ומגדיר את ה-runner אוטומטית כשירות Windows.
💡 לאחר ההגדרה, הסקריפט מגדיר את Execution Policy ומפעיל את השירות.
6 גיבוי Google Drive
העתק את rclone.conf מהמחשב הישן חד-פעמי
אם אתה משתמש באותו חשבון Google Drive — פשוט העתק: 
# מהמחשב הישן:
C:\Users\davidferber\AppData\Roaming\rclone\rclone.conf

# לנתיב הזהה בשרת החדש
💡 אם לא — הרץ C:\actions-runner\rclone.exe config ועקוב אחר תהליך Google Auth.
הגדר גיבוי אוטומטי אוטומטי
הסקריפט setup.ps1 מעתיק את backup.ps1 ויוצר Task Scheduler אוטומטית.
לבדיקה ידנית: 
powershell -ExecutionPolicy Bypass -File C:\actions-runner\backup.ps1
7 בדיקות סיום
האתר עולה
פתח http://<LAN_IP> — מסך הכניסה אמור להופיע.
כניסה עם משתמש עובד
נסה להתחבר עם אחד מהמשתמשים הקיימים.
Auto-deploy עובד
בצע שינוי קטן ו-git push — בדוק שה-GitHub Actions run מסתיים בהצלחה ב: 
https://github.com/workdavid353-oss/erez-management/actions
גיבוי ידני עובד
הרץ גיבוי ידני ובדוק שהקובץ מופיע ב-Google Drive: 
powershell -ExecutionPolicy Bypass -File C:\actions-runner\backup.ps1
📋 עזר מהיר — קבצים חשובים

C:\actions-runner\.env.productionעותק עבור GitHub Actions runner
C:\actions-runner\backup.ps1סקריפט גיבוי
C:\actions-runner\rclone.exeכלי העלאה ל-Google Drive
C:\actions-runner\backups\backup.logלוג גיבויים
supabase\docker\.envהגדרות Supabase (לא ב-git)
